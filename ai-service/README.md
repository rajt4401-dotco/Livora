# Livora AI Service

A Flask-based ML microservice providing intelligent features for the Livora Hostel Management System.

## Features

| Feature | Model | Endpoint |
|---|---|---|
| Complaint Classification | Logistic Regression + TF-IDF | `POST /analyze` |
| Priority Detection | Random Forest + TF-IDF | `POST /analyze` |
| Sentiment Analysis | Logistic Regression + TF-IDF | `POST /sentiment` |
| Room Allocation | K-Means Clustering (k=3) | `POST /room-allocation` |
| Fee Default Prediction | Random Forest | `POST /predict-fee` |

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Train models (first time only)
```bash
python train.py
```
> Models are saved to `models/saved/` as `.pkl` files.

### 3. Start the server
```bash
python app.py
```
> Server runs on `http://localhost:8000`

**Windows:** double-click `start.bat` — it auto-trains and starts the server.

## API Endpoints

### `GET /health`
Returns service status and which models are loaded.

### `POST /analyze`
Full complaint analysis — classification + priority + sentiment in one call.
```json
{ "text": "water leakage in bathroom urgent" }
```
Response:
```json
{
  "success": true,
  "category": "Plumbing",
  "priority": "High",
  "sentiment": "negative",
  "confidence": { "category": 0.94, "priority": 0.87, "sentiment": 0.81 }
}
```

### `POST /classify`
Category classification only.
```json
{ "text": "wifi not working" }
```

### `POST /sentiment`
Sentiment analysis only.
```json
{ "text": "food quality has improved a lot" }
```
Returns: `positive | negative | neutral`

### `POST /predict-fee`
Predict if a student will default on fees.
```json
{
  "unpaid_semesters": 2,
  "late_payments": 3,
  "year": 3,
  "course_type": 0,
  "part_time_job": 0,
  "scholarship": 0
}
```
Returns: `riskLevel: High | Medium | Low`

### `POST /room-allocation`
Cluster students into compatible room groups.
```json
{
  "students": [{
    "id": "s1",
    "name": "Rahul",
    "preferences": {
      "night_owl": 0,
      "cleanliness_score": 9,
      "noise_tolerance": 2,
      "study_hours": 9,
      "social_score": 2
    }
  }],
  "rooms": [{ "id": "r1", "number": 101, "status": "Available" }]
}
```

### `POST /sentiment/batch`
Analyze multiple texts at once (max 50).
```json
{ "texts": ["wifi is great!", "food is terrible", "room is okay"] }
```

```
ai-service/
 ├── app.py              <- Flask API server
 ├── train.py            <- Model training pipeline
 ├── test_api.py         <- API test suite
 ├── start.bat           <- Windows startup script
 ├── requirements.txt
 ├── data/
 │    ├── complaints.csv  <- 90+ labeled complaint samples
 │    ├── priority.csv    <- 40+ priority-labeled samples
 │    ├── sentiment.csv   <- 40+ sentiment-labeled samples
 │    ├── room_allocation.csv
 │    └── fees.csv
 └── models/
      ├── registry.py     <- Singleton model loader
      └── saved/          <- Trained .pkl files (generated by train.py)
```

## Model Details

| Model | Algorithm | Training Samples | Notes |
|---|---|---|---|
| Complaint Classifier | LR + TF-IDF (1,2)-grams | 90 | 9 categories |
| Priority Detector | Random Forest + TF-IDF | 41 | 3 levels |
| Sentiment Analyser | LR + TF-IDF (1,2)-grams | 40 | 3 classes |
| Room Allocator | K-Means (k=3) | 20 student profiles | 5 lifestyle features |
| Fee Predictor | Random Forest | 20 records | 6 features |

All models have **keyword-based fallbacks** — the service works even without MongoDB or the trained `.pkl` files.

## Connecting to Backend

The Node.js backend at `http://localhost:5000` calls this service via `aiService.js`. No configuration needed — it auto-connects to `http://localhost:8000`.
